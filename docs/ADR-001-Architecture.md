# ADR-001: Выбор архитектуры и стэка для VPN-бота

## Статус
Принято

## Контекст
Проекту требовалась высокая отказоустойчивость, способность обрабатывать пиковые нагрузки при оплатах и не зависеть от временной недоступности панелей 3x-ui.

## Решение
1. **Чистая архитектура (Clean Architecture) + DI**: Использование `dependency-injector` для разделения слоев.
2. **Асинхронность**: Aiogram 3 + aiohttp + aiosqlite.
3. **Фоновые задачи (Event-Driven)**: Внедрение `arq` + `Redis` для тяжелых задач (бэкапы, массовые уведомления), чтобы бот оставался отзывчивым.
4. **Миграции**: Использование `Alembic` для безопасного изменения схемы SQLite в production.
5. **Observability**: `structlog` (JSON) + OpenTelemetry + Prometheus метрики.

## Последствия
* (+) Бот не блокируется при сетевых задержках 3x-ui.
* (+) Безопасный деплой обновлений без потери данных.
* (-) Усложнение инфраструктуры (необходим Redis и Docker-compose).
